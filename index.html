##Anatomy of a Web Page **HTML (Hypertext Markup Language)** is used to define the meaning of the content, known as [semantics](http://learn.shayhowe.com/html-css/getting-to-know-html/) and to create the structure of the page. **CSS (Cascading Style Sheets)** is the presentation layer used to define how the page will look (e.g. colours, fonts, etc). **JavaScript** is a programming language. This is usually used to defines the behaviour and interactions. (e.g. when the user scrolls down the page, do something) --- ###HTML + CSS + JavaScript = Front-end development / Client-side
#HTML ##The content layer
##HTML / XHTML / HTML5 What's the difference? **HTML (HyperText Markup Language)**: Used to structure and define page content. **XHTML (Extensible HyperText Markup Language)**: Reformed HTML based on XML. Syntax rules are more strict. **HTML5 (HyperText Markup Language, revision 5)**: Most current version. Improved the support for multimedia and introduced new structural tags. Not supported in older browsers. Basically, these three terms refer to different versions of the same language but are often just called HTML interchangeably. The [World Wide Web Consortium](http://www.w3.org/TR/html5/) (W3C) governs these web standards with specifications, guidelines, tools and recommendations.
##HTML Terminology **Elements:** Components that define structural objects such as paragraphs (`p`). **Tags:** Used to define HTML elements. Denoted by `<`angled brackets`>` (`shift` + `,` and `shift` + `.` ) Usually come in pairs (but not always) that open to mark the beginning of an element and close the element with a forward slash. ``` <tag>content here</tag> ``` ``` <p>I'm a paragraph.</p> ``` Tags can also be nested inside other tags. Just make sure they always close in the reverse order. ``` <p>I am a paragraph with <strong>bold text</strong>.</p> ``` **Attributes:** provides additional information or instruction for the element and is included *inside* the opening HTML tag in the form of `attribute="value"`. More on this in the next slide.

##Good File Naming Habits * Use lowercase letters. (e.g. about.html instead of About.html or ABOUT.html) * Avoid s p a c e s. Underscores (_) or dashes (-) are valid. However, dashes are generally preferred for SEO (search engine optimization). (e.g. business-hours.html instead of business hours.html - The browser will interpret it as business%20hours.html) * Avoid symbols like # & * in your file name. * Keep your file names concise, yet meaningful. (e.g. about.html, contact.html instead of page1.html, page2.html) These file naming tips can be applied to any files such as images, folders, CSS, etc.
##Adding Content: Headings & Paragraphs There are six headings tags ranging from `<h1>` to `<h6>` use to create page headings. Some tips to follow: * describes the hierarchy of the page and gives content meaning and structure (semantics) * use specific headings tags to describe the content, not based on the size of the default styles * use CSS to style the headers accordingly * makes is easier for screen readers, search engines to read and understand ```xml <h1>Name of blog</h1> <h2>Title of blog post</h2> <h3>Sub headings within blog post</h3> ``` Paragraphs are represented by the `<p>` tag. ```xml <p>This is a paragraph.</p> <p>This is another paragraph.</p> ```
